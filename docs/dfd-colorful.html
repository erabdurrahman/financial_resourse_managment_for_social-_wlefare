Process & Data Store Descriptions

ID Name Description
P1 User Authentication Validates credentials, registers new users, issues JWT tokens, enforces role-based access (admin / donor / beneficiary).
P2 Donation Management Accepts donation submissions from verified donors, stores records in D2, computes total donated funds.
P3 Application Management Validates & stores assistance applications in D3; calculates a server-side priority score (0–100) from income, family size, urgency, and documents.
P4 Fund Allocation & Review Admin approves or rejects pending applications; approved applications generate allocation records in D4, reducing available balance.
P5 Reporting & Statistics Aggregates D2 + D4 to produce role-specific dashboards: Admin sees all stats; Donor sees own donation history; Beneficiary sees application status & score.
D1 USERS table id · name · email · password_hash · role (admin | donor | beneficiary)
D2 DONATIONS table id · donor_id · amount · message · created_at
D3 APPLICATIONS table id · beneficiary_id · income · family_members · urgency · amount · category · reason · priority_score · priority_level · status · documents_path
D4 TRANSACTIONS table id · application_id · donor_id · amount · type='allocation' · created_at
